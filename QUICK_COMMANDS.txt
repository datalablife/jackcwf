🚀 PUSH TO GITHUB (Choose one method)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  METHOD 1: GitHub CLI (Recommended)
    gh auth login
    cd /mnt/d/工作区/云开发/working
    git push -u origin main

  METHOD 2: Windows Git (Avoids WSL issues)
    cd "D:\工作区\云开发\working"
    git push -u origin main

  METHOD 3: HTTPS
    cd /mnt/d/工作区/云开发/working
    git push -u origin main
    # Enter GitHub token when prompted

  METHOD 4-7: See PUSH_TO_GITHUB.md for other methods

⚙️  CONFIGURE GITHUB SECRETS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Automated setup:
    cd /mnt/d/工作区/云开发/working
    ./scripts/ci/setup-secrets.sh

  Manual setup:
    1. Go to: https://github.com/datalablife/jackcwf/settings/secrets/actions
    2. Add these secrets:
       - COOLIFY_API_TOKEN
       - COOLIFY_SERVER_URL

🔧 ENABLE GITHUB ACTIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Go to: Settings → Actions → General
  2. Select: "Allow all actions and reusable workflows"
  3. Click: Save

🆘 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Push fails with network error:
    • Use GitHub CLI: gh auth login; git push
    • Or use Windows Git from Windows host
    • See PUSH_TO_GITHUB.md for WSL fixes

  Workflows not running:
    • Check if GitHub Actions is enabled (see above)
    • Check if main is the default branch
    • Look for red errors in Actions tab

  Deployment fails:
    • Check if secrets are configured
    • Verify Coolify API token is valid
    • Check deployment logs in Actions

  See PUSH_TO_GITHUB.md for more troubleshooting

🎯 TYPICAL WORKFLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Push code to GitHub:
       git push -u origin main

  2. Verify on GitHub:
       Check https://github.com/datalablife/jackcwf

  3. Configure secrets:
       ./scripts/ci/setup-secrets.sh

  4. Enable GitHub Actions:
       Settings → Actions → Allow all

  5. Test CI with PR:
       Create test branch and PR
       Watch Actions for CI run

  6. Test deployment:
       Actions → CD workflow → Run workflow
       Select environment: staging

  7. Deploy to production:
       Actions → CD workflow → Run workflow
       Select environment: production
       Approve deployment
